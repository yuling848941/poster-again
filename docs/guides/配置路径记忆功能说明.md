# 配置路径记忆功能说明

## 📋 功能概述

为提升用户体验，"保存当前配置"和"加载配置文件"功能现在可以记住上次选择的目录路径，无需每次都重新浏览查找。

## ✨ 功能特性

### 1. 路径记忆
- **保存配置**: 自动记住上次保存配置的目录
- **加载配置**: 自动记住上次加载配置的目录
- **持久化存储**: 路径保存在 `config/config.yaml` 中，程序重启后仍然有效

### 2. 智能初始化
- **首次使用**: 首次使用时默认打开用户主目录
- **后续使用**: 自动打开上次使用的目录
- **自动保存**: 选择路径后自动更新记录

### 3. 兼容性保证
- **向下兼容**: 不影响现有配置文件格式
- **向后兼容**: 旧版本配置可正常使用
- **路径可选**: 如果保存的路径不存在，自动回退到主目录

## 🔧 技术实现

### 修改的文件

1. **src/config_manager.py**
   - 在 `default_config["paths"]` 中添加了：
     - `last_config_save_dir`: 记住保存配置的最后目录
     - `last_config_load_dir`: 记住加载配置的最后目录

2. **src/gui/config_dialog.py**
   - 修改 `ConfigSaveLoadDialog` 类：
     - 使用全局 `config_manager` 实例
     - `save_config()` 方法：从配置中读取保存目录
     - `load_config()` 方法：从配置中读取加载目录
     - 保存/加载成功后自动更新目录记录

3. **main.py**
   - 添加全局 `config_manager` 实例
   - 配置管理器在程序启动时初始化

### 核心代码逻辑

#### 读取保存的路径
```python
last_save_dir = self.config_manager.get("paths.last_config_save_dir", "")
if not last_save_dir:
    last_save_dir = os.path.expanduser('~')  # 默认主目录

file_path, _ = QFileDialog.getSaveFileName(
    self,
    "保存配置文件",
    last_save_dir,  # 使用记住的目录
    "配置文件 (*.pptcfg);;所有文件 (*.*)"
)
```

#### 更新保存的路径
```python
save_dir = os.path.dirname(file_path)
self.config_manager.set("paths.last_config_save_dir", save_dir)
self.config_manager.save_config()  # 持久化保存
```

## 🎯 使用场景

### 场景1: 频繁保存配置
**用户**: 每次修改配置后都要保存
**问题**: 每次都要点击"浏览"选择目录
**解决**: 首次选择后，后续自动打开该目录

### 场景2: 团队共享配置
**用户**: 团队成员共享配置文件
**问题**: 每个人的配置文件放在不同目录
**解决**: 程序记住各自的配置目录，互不干扰

### 场景3: 分类管理
**用户**: 按项目分类保存配置文件
**问题**: 需要记住每个项目的配置目录
**解决**: 程序自动记住最近使用的目录
