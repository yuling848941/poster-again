# 占位符问题诊断指南

## 问题描述

您在PPT模板中明明设置了前缀为"ph_"的占位符，但程序自动匹配时提示"模板中未找到占位符"。

## 常见原因

### 1. 形状名称不匹配
- **问题**：在PowerPoint选择窗格中看到的名称可能与实际形状的`Name`属性不同
- **解决方案**：需要确保形状的`Name`属性确实以"ph_"开头

### 2. 选择了错误的形状类型
- **问题**：选中的可能是图片、线条等非文本形状
- **解决方案**：确保选择的是文本框、占位符或可编辑的文本形状

### 3. 名称中包含隐藏字符
- **问题**：名称中可能有空格、特殊字符等
- **解决方案**：清理名称，只保留字母、数字和下划线

## 诊断方法

### 方法一：使用调试工具

运行调试工具查看PPT中的实际形状名称：

```bash
python debug_placeholders.py
```

输入您的PPT文件路径（如 `E:/poster code/poster-again/10月承保贺报(4).pptx`），工具将显示：

- 所有形状的名称
- 形状类型（文本框、图片等）
- 哪些形状被识别为占位符

### 方法二：手动检查PowerPoint

1. **打开PPT模板**
2. **按Alt+F10打开选择窗格**
3. **逐一检查每个形状的名称**
4. **确保以"ph_"开头的形状**：
   - 是文本形状（文本框、占位符）
   - 名称确实是 `ph_xxx` 格式

## 正确设置占位符的步骤

### 步骤1：选择形状
确保您选择的是：
- ✅ 文本框
- ✅ 占位符
- ❌ 图片
- ❌ 线条
- ❌ 组合形状（需要先取消组合）

### 步骤2：设置名称

**方法一：使用选择窗格**
1. 选中形状
2. 在选择窗格中，右键点击形状名称
3. 选择"重命名"
4. 输入如 `ph_姓名`、`ph_日期` 等

**方法二：使用编辑Alt文本**
1. 选中形状
2. 右键 -> "编辑 Alt 文本"
3. 在"Alt 文本"窗口中，可能有"名称"字段

### 步骤3：验证名称
- 名称必须以 `ph_` 开头
- 名称中不能包含空格（可以使用下划线替代）
- 名称示例：
  - ✅ `ph_姓名`
  - ✅ `ph_日期`
  - ✅ `ph_公司名称`
  - ❌ `ph 姓名`（包含空格）
  - ❌ `姓名`（缺少ph_前缀）

## 常见错误示例

### 错误1：名称显示正确但实际不正确
```
选择窗格显示：文本框 1
实际Name属性：TextBox 1
```
**解决方案**：重命名形状为 `ph_文本框1`

### 错误2：选择了图片但误以为是文本框
```
形状类型：图片 (Picture)
名称：ph_图片
```
**解决方案**：删除图片，添加文本框并命名为 `ph_xxx`

### 错误3：名称包含中文字符
```
名称：ph_姓名
```
**解决方案**：可以保留，但建议使用英文：`ph_name`

## 验证占位符是否正确

运行调试工具后，您应该看到：

```
【幻灯片 1】
------------------------------------------
  1. 形状名称: ph_姓名 [占位符]
     类型: 文本框
     文本内容: '请输入姓名'
  2. 形状名称: ph_日期 [占位符]
     类型: 文本框
     文本内容: '请输入日期'
```

如果没有看到 `[占位符]` 标记，说明名称设置不正确。

## 解决步骤总结

1. **运行调试工具**：`python debug_placeholders.py`
2. **查看实际形状名称**
3. **在PowerPoint中重新命名所有需要作为占位符的形状**
4. **重新保存PPT模板**
5. **再次运行程序测试**

## 注意事项

- 每次修改形状名称后，需要保存PPT文件
- 确保所有占位符都是可编辑的文本形状
- 建议占位符名称简洁明了，便于理解用途
- 程序会自动匹配 `ph_` 开头的形状，无需其他特殊设置

如果按照以上步骤操作后仍有问题，请提供调试工具的输出结果，以便进一步诊断。
