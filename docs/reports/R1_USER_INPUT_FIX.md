# R=1情况用户输入修复总结

## 问题描述

当R=1时，用户通过对话框输入数值后，批量生成PPT时对应元素无显示。

## 修复内容

### 1. 字符串处理修复
**文件**: `src/memory_management/data_formatter.py`
**修改**: 添加千位分隔符字符串清理逻辑
```python
# 转换为数值类型（处理可能包含千位分隔符的字符串）
if isinstance(sfyp2_value, str):
    sfyp2_clean = sfyp2_value.replace(',', '').replace(' ', '')
    sfyp2_num = float(sfyp2_clean)
```

### 2. GUI交互优化
**文件**: `src/gui/main_window.py`
**修改**:
- 添加"承保趸期数据"菜单选项
- 优化用户输入验证逻辑
- 保存用户输入数据到主窗口状态

### 3. PPT生成时数据处理
**文件**: `src/gui/ppt_worker_thread.py`
**修改**: 在批量生成前重新计算承保趸期数据

## 已知问题与解决方案

### 问题1: 用户输入数据传递
**现象**: 对话框输入的数据未传递给PPT生成过程
**原因**: 用户输入数据仅保存在DataReader内存中，PPT生成时重新加载原始Excel
**解决方案**: 
1. 将用户输入数据保存到临时文件
2. 或在PPT生成时重新计算（包括用户输入）

### 问题2: 占位符名称匹配
**现象**: 即使数据正确，占位符也可能无法显示
**原因**: 占位符名称与列名匹配问题
**解决方案**: 
- 确保PPT中的占位符格式正确：`ph_字段名`
- 在GUI中正确选择要关联的占位符

1. **占位符命名**: 必须使用`ph_`前缀，如`ph_test`
2. **列名匹配**: 确保Excel包含`SFYP2(不含短险续保)`和`首年保费`列
3. **数据格式**: 数值列不应包含非数字字符（除千位分隔符外）
4. **对话框输入**: 必须填写所有需要输入的行
