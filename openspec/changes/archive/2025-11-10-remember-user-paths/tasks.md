# 记住用户路径和设置选择功能 - 实现任务

## 实现任务清单

### 1. ConfigManager功能增强
- [x] 1.1 添加 `load_last_paths()` 方法
  - 从配置中加载上次保存的路径
  - 返回包含所有路径类型的字典
  - 处理配置项不存在的情况

- [x] 1.2 添加 `get_start_dir_for_file_type(file_type)` 方法
  - 根据文件类型返回对应的起始目录
  - 支持的文件类型：template, data, output
  - 处理无效或不存在路径的情况

- [x] 1.3 添加 `validate_path(path)` 方法
  - 检查路径是否存在
  - 验证路径的可访问性
  - 返回布尔值表示路径有效性

- [x] 1.4 增强 `update_last_paths()` 方法
  - 添加路径验证逻辑
  - 改进错误处理
  - 添加日志记录

### 1.5 图片生成设置管理 (NEW)
- [x] 1.5.1 添加 `load_image_generation_settings()` 方法
  - 从配置中加载图片生成设置

  - 处理配置项不存在的情况，使用默认值

- [x] 1.5.2 添加 `update_image_generation_settings()` 方法

  - 添加参数验证（enabled为布尔值，format为PNG/JPG，quality为有效数值）
  - 添加日志记录

## 验收标准

1. **功能验收**：
   - 程序启动时能正确恢复上次路径
   - 文件选择器从正确目录开始
   - 路径选择后能自动保存
   - 程序启动时能正确恢复图片生成设置
   - 图片生成设置修改后能自动保存
   - 图片生成控件状态正确恢复

2. **质量验收**：
   - 所有测试通过
   - 无内存泄漏
   - 异常情况处理正确
   - 图片设置验证正常工作

3. **用户体验验收**：
   - 路径恢复过程对用户透明
   - 无效路径时程序仍能正常工作
   - 图片生成设置恢复过程对用户透明
   - 无效设置时程序使用默认值
   - 功能不影响现有操作流程
