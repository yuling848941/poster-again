# 承保趸期数据功能提案

## 概述

本提案旨在为PPT批量生成工具添加"承保趸期数据"功能，该功能类似现有的"递交趸期数据"功能，但使用不同的数据源和计算逻辑，以支持更复杂的承保场景。

## 问题背景

现有系统中的"递交趸期数据"功能基于"缴费年期（主险） ID"列进行计算，这种方式适用于简单的缴费年期场景。但对于承保场景，需要基于"SFYP2(不含短险续保)"和"首年保费"两个财务指标进行更复杂的计算，以生成更准确的趸期数据。

承保场景的特点：
1. 数据源更复杂，需要两个财务指标进行比例计算
2. 计算结果可能需要人工输入特定数值
3. 需要批量处理多个行并支持一次性输入
4. 输出格式需要与现有系统保持一致

## 解决方案

通过扩展数据处理管道，添加"承保趸期数据"计算功能：

1. **数据检查**：检测Excel中是否存在"SFYP2(不含短险续保)"和"首年保费"列
2. **比例计算**：计算R = SFYP2 / 首年保费
3. **分类处理**：
   - R=0.1：直接输出"趸交FYP"
   - R=1：弹对话框让用户输入正整数
   - R=0.2~0.9：计算R×10并输出格式化的年期
4. **批量输入界面**：提供对话框让用户一次性填写所有需要手工输入的数值
5. **数据存储**：将结果存储在"承保趸期数据"列中

## 预期收益

1. **扩展功能覆盖**：支持承保场景的趸期数据处理
2. **提高自动化程度**：减少手工计算和输入工作量
3. **改善用户体验**：批量输入界面简化操作流程
4. **保持一致性**：与现有"递交趸期数据"功能保持一致的使用体验

## 技术风险与缓解

1. **风险**：用户输入验证复杂性
   - 缓解：严格的输入验证规则，只接受有效的正整数

2. **风险**：数据格式兼容性
   - 缓解：在计算前进行充分的数据验证和类型检查

3. **风险**：性能影响（大量数据行）
   - 缓解：优化批量处理逻辑，只在必要时触发用户交互

## GUI实现与集成细节

### 主窗口集成（src/gui/main_window.py）

1. **菜单扩展**：
   - 在右键菜单中添加"承保趸期数据"选项
   - 连接`submit_chengbao_term_data()`方法

2. **数据处理流程**：
   - 调用DataReader加载数据并处理承保趸期数据
   - 检查是否有R=1的行需要用户输入
   - 弹出对话框收集用户输入

3. **数据同步**：
   - 在关联前确保PPTGenerator已加载包含承保趸期数据的数据
   - 保存用户输入数据到MainWindow.chengbao_user_inputs属性

4. **占位符关联**：
   - 调用PPTGenerator.add_matching_rule()关联占位符与数据列
   - 更新worker_thread中的匹配规则

### PPTWorkerThread增强（src/gui/ppt_worker_thread.py）

1. **构造函数修改**：
   - 新增main_window参数
   - 保存MainWindow引用以访问用户输入数据

2. **数据处理增强**：
   - 从ppt_generator.data_reader获取数据（而非ppt_generator.data）
   - 检查承保趸期数据列存在性
   - 应用保存的用户输入数据到DataFrame

3. **验证机制**：
   - 验证用户输入应用结果
   - 检查匹配规则与数据的匹配性
   - 记录详细的调试日志

### 关键修复

1. **数据引用修正**：
   - 问题：PPTWorkerThread访问了错误的数据路径
   - 解决：使用ppt_generator.data_reader而非ppt_generator.data

2. **占位符关联时机**：
   - 问题：关联时ppt_generator没有数据
   - 解决：在关联前加载数据到ppt_generator

3. **用户输入传递**：
   - 问题：用户输入数据无法传递到PPT生成
   - 解决：通过MainWindow状态保存和工作线程引用传递
